## Overview

**TrueImage** is a deep learning-based web system designed to detect whether a submitted human face image is likely to be real or AI-generated.

The system focuses on still images of human faces and provides a confidence-based result rather than relying only on a simple real/fake label. It is designed with privacy, usability, and practical deployment in mind, making it suitable for academic research, demonstrations, and further development in digital image verification.

---

## Project Objectives

The main objectives of TrueImage are to:

- Detect AI-generated human face images using a trained deep learning model.
- Validate uploaded images before classification to ensure that a human face is present.
- Provide a user-friendly web interface for uploading and analyzing images.
- Return a confidence score that helps users interpret the prediction.
- Temporarily process uploaded images without permanently storing user data.

---

## Key Features

### AI-Generated Face Detection

TrueImage uses a trained deep learning model to analyze facial image patterns and estimate the likelihood that an image was generated by artificial intelligence.

### Face Validation

Before classification, the system checks whether the uploaded image contains a detectable human face. This helps prevent non-face images, diagrams, or unrelated uploads from being passed directly to the classification model.

### YuNet-Based Face Detection

The project includes the **YuNet ONNX face detection model**, which is used with OpenCV for efficient face localization.

### Confidence-Based Output

Instead of only displaying a binary label, the system presents the prediction as a confidence score, allowing the user to understand how strongly the model supports its result.

### Privacy-Focused Processing

Uploaded images are handled as temporary files during analysis. The system is designed to avoid unnecessary long-term storage of user images.

## Installation and Setup

### 1. Clone the Repository

```bash
git clone https://github.com/ShekinahBeliaMulenga/trueimage.git
cd trueimage
```

### 2. Create a Virtual Environment

#### Windows

```bash
python -m venv venv
venv\Scripts\activate
```

#### macOS / Linux

```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

> If `requirements.txt` has not yet been fully updated, install the required packages manually and then freeze them into the file using:
>
> ```bash
> pip freeze > requirements.txt
> ```

### 4. Run the Application

```bash
python run.py
```

The application will start on:

```text
http://localhost:5000
```

If running on a local network, the app may also be accessible through the host machine’s local IP address.

## How the System Works

The system follows a simple analysis pipeline:

1. **Image Upload**
   The user uploads an image through the web interface.

2. **Input Validation**
   The system checks whether the file is valid and can be processed as an image.

3. **Face Detection**
   OpenCV and YuNet are used to detect whether the image contains a human face.

4. **Preprocessing**
   The detected face region is prepared for model inference.

5. **AI-Generated Image Classification**
   The deep learning model analyzes the image and produces a confidence score.

6. **Result Display**
   The prediction result is displayed to the user in a simple and understandable format.

7. **Temporary File Handling**
   Uploaded files are treated as temporary processing artifacts.

## Limitations

TrueImage is a research and academic system. Its results should be interpreted carefully.

Current limitations may include:

* The system focuses on still human face images only.
* It does not detect AI-generated videos or audio.
* Very low-quality, blurred, edited, or heavily compressed images may affect accuracy.
* The model may produce false positives or false negatives.
* Results should not be used as the only basis for high-stakes decisions.

## Security and Privacy Considerations

TrueImage is designed with privacy in mind. Uploaded images should only be used for the analysis process and should not be stored permanently unless explicitly required for research or debugging.

Recommended privacy practices include:

* Automatically deleting temporary uploads.
* Avoiding unnecessary logging of user image data.
* Restricting upload file types and file sizes.
* Keeping trained models and dependencies updated.
* Using HTTPS in production deployments.
